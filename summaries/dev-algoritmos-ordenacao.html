Conceitos Fundamentais

Bases Teóricas Exigidas em Prova

Estabilidade

Um algoritmo é considerado estável quando preserva a ordem relativa de elementos com chaves iguais (duplicadas) durante a ordenação.

Estáveis: BubbleSort, InsertionSort e MergeSort.
Instáveis: SelectionSort, QuickSort, HeapSort e ShellSort.

Notações Assintóticas

Determinam o custo computacional independentemente do hardware, medindo as operações primitivas em função do tamanho (n) da entrada.

  • Big-O (O): Representa o Pior Caso (limite superior). É a mais cobrada em provas.
  • Big-Ômega (Ω): Representa o Melhor Caso (limite inferior).
  • Big-Theta (Θ): Representa o Caso Médio (limite exato).

Dica de Prova

Na notação assintótica, a base do logaritmo (seja base 2, 10 ou neperiano) é irrelevante, pois é tratada matematicamente como uma constante.

Ordem de Crescimento (Mais rápido → Mais lento)

O(1) < O(log n) < O(n) < O(n log n) < O(n2) < O(n3) < O(2n) < O(n!)

Algoritmos de Ordenação

Associações Rápidas para a Prova

Bubble Sort

Bolha / Troca

Move elementos para posições adequadas trocando pares consecutivos que estão fora de ordem. Simples, porém pouco eficiente para grandes listas.

Insertion Sort

Inserção

Insere cada elemento em sua posição correta dentro de um subvetor já classificado à esquerda. Muito eficiente para listas que já estão quase ordenadas ou pequenas.

Selection Sort

Seleção

Seleciona o menor elemento do vetor e o troca com o primeiro elemento da parte não ordenada. Seu ponto forte é realizar poucas operações de troca (swap).

Shell Sort

Inserção

Extensão e refinamento do InsertionSort. Utiliza uma distância (gap) para comparar e realizar trocas, diminuindo até 1. O mais eficiente entre os de ordem quadrática.

Quick Sort

Troca / Divisão-e-Conquista

Elege um pivô e particiona o vetor. Muito rápido na prática, mas seu pior caso ocorre quando o pivô escolhido é um dos extremos.

Merge Sort

Intercalação / Divisão-e-Conquista

Divide a lista na metade recursivamente e depois as intercala em ordem. Tempo sempre O(n log n), garantindo consistência.

Heap Sort

Seleção

Utiliza uma estrutura de árvore chamada heap. Vantagem: estrutura auxiliar de armazenamento fora do arranjo constante (exige pouco espaço extra).

Algoritmos de Busca (Pesquisa)

Eficiência: de um em um vs. Dividir para Pesquisar

Busca Sequencial (Linear)

Inspeciona posição por posição desde o início até achar o elemento ou chegar ao fim do vetor.

  • Pré-requisito: Funciona em vetores ordenados ou desordenados.
  • Uso: Inaceitável para grandes volumes de dados, útil em listas pequenas e mutáveis.
Melhor (Início): O(1)
Pior / Médio: O(n)

Busca Binária

Usa Divisão-e-Conquista. Compara com o elemento central e descarta metade do vetor a cada iteração.

  • Pré-requisito Obrigatório: O vetor precisa estar ORDENADO.
  • Uso: Muito mais eficiente e rápida que a Linear para grandes coleções.
Pior / Médio: O(log2 n)

Dica de Revisão Final

Concentre-se em não confundir os algoritmos estáveis com os instáveis e atente-se ao calcanhar de aquiles do QuickSort (seu pior caso ser O(n2), enquanto Merge e Heap garantem O(n log n)).
